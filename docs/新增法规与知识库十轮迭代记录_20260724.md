# 新增法规与知识库十轮迭代记录

日期：2026-07-24

本轮采用“小改动—局部验证—全量重建”的方式，把补充法规导入、正文清洗、Chunk、检索索引、静态阅读页、分类证据表和真实问答链路放在同一条可复核流水线上。最终有效法规为216部，另有2个已嵌入主文件的附件原件仅归档、不独立参与检索。

## 第1轮：原件与清单盘点

- 对照补充清单逐行核对标题、日期、发文主体、效力状态和官网链接。
- 补齐清单缺失的第4、6项元数据，并从深交所官网下载缺少的第14项官方 PDF。
- 识别出“可投资国家或者地区”“期货期权交易所”是主法规内嵌附件，保留原件但排除独立入库，避免重复召回。

验证：102部补充法规、2个内嵌附件全部找到唯一原件；标题匹配最低分0.7586。

## 第2轮：归档与目录导入

- 将补充 Word、PDF、DOC 原件移入 `data/raw/监管文件`。
- 新增 `scripts/import_supplemental_regulations.py`，以清单为元数据源，生成可重复执行的目录和导入审计。
- 导入脚本重跑时保留已经生成的 Document ID、Chunk 数、字符数和版本信息。

验证：目录共218条，其中216条参与处理；脚本幂等性检查通过。

## 第3轮：封面与附件清洗

- 修复独立“附件”字样、重复标题、版本日期封面残留。
- 保留有实质内容的附件标题与附件正文，避免把法规组成部分误删。
- 兼容旧 DOC 前缀中的孤立“目录”。

验证：用户指出的三类上海证券交易所文件封面均不再混入正文首段。

## 第4轮：PDF 与字符清洗

- 修复 PDF 正文中条款引用被错误断开的情况。
- 清理已核实无语义的私用区字符，同时保留数学符号、公式和有意义的版式内容。
- 更新解析与清洗版本，确保产出能够追溯到规则版本。

验证：全量解析216/216成功。

## 第5轮：结构识别与 Chunk

- 修正“（二）短时间内报单……达到”等规范性列项被误判为标题的问题。
- 继续以条、章、节和附件为边界切分，并保留条号、章节路径、页码和块定位。
- 对结构化文档与 Chunk 执行全量质量检查。

验证：生成2,642个 Chunk；0个严重问题、0个主要问题，仅保留64个不影响使用的次要提示；Python 测试85项通过。

## 第6轮：BM25 与向量索引

- 基于同一份2,642 Chunk语料重建 BM25、向量矩阵、文档元数据和索引清单。
- 记录逐行嵌入输入哈希和向量哈希，检查语料行号与向量行号严格一致。
- 增加无向量重建的写前兼容性检查，避免失败后留下“新语料＋旧向量”的半更新索引。

验证：216部文档、6,561个嵌入片段、68,998个BM25词项；重复正文审计为0；检索测试12项通过。

## 第7轮：真实 Agent 问答

- 新增3道覆盖补充法规的通用测试题，分别验证高频交易报告、信息隔离墙回墙条件、可转债与可交换债交易限制。
- 通过独立4001端口真实调用 API，不使用缓存答案。

验证：3/3回答通过，引用 ID、法规依据和引用校验结构均有效；结果保存在 `data/index/eval/live-results/new-supplemental-20260724.json`。

## 第8轮：法规阅读 HTML

- 以216部规范化正文重新生成单文件阅读页，并同步旧法规的最新清洗结果。
- 官网原文链接全部改为目录中的官方链接；“分类详情”改为“阅读详情”。
- 每页由8部改为7部；优化正文层级、附件、列表、表格和阅读元信息展示。

验证：浏览器实测搜索、筛选、分页、阅读正文和官网链接正常；首页严格显示7部法规。

## 第9轮：分类映射与 Excel

- 使用完整正文按“适用对象、业务类型、监管事项”三维重新计算标签。
- 每个标签保留标注理由、命中词、条章/页码/Block ID、原文摘录、规则分和官网链接。
- 生成四表 Excel：使用说明、分类标准、法规标签总览、标签证据明细。

验证：216部法规、2,303条标签证据；工作簿压缩结构检查通过，四个工作表均完成渲染复核。

## 第10轮：整体审查与交付

- 审查解析、Chunk、索引、Agent、HTML、分类、Excel 和导入脚本之间的数据一致性。
- 修复索引构建的事务安全和补充导入脚本的幂等性。
- 执行 Python、检索、API 合约、TypeScript/构建及产物一致性检查。

回滚方式：本轮所有变更由单个 Git 提交管理；需要回退时对该提交执行 `git revert`，原始监管文件仍保留在 `data/raw/监管文件`，不会因回滚产出而丢失。
