# 法规正文与增量构建验证报告

日期：2026-07-19

## 结果摘要

当前正式知识库包含 114 部法规、1723 个检索 Chunk 和 1723 行向量。HTML 阅读正文的数据源已统一为结构化文档中的 `clean_text + structured_blocks`，不再从带 overlap 的 Chunk 反向拼接全文。

本轮 18 部法规的清洗后正文发生变化，96 部法规复用既有 Chunk。向量构建按 embedding 输入哈希复用 1152 行旧向量，新增或重算 571 行；`data/index/incremental_vector_audit.csv` 证明 1152 个复用行的前后向量 SHA-256 完全一致。

## 数据链路

```text
原始 PDF / Word / HTML
  → 原始块与原生表格网格
  → front matter、正文、附件分类
  → clean_text + structured_blocks
  ├─→ 带位置及 overlap 的检索 Chunk → BM25 / embedding
  └─→ 单文件 HTML 阅读原文
```

- `front_matter` 保存发布包装、独立文号、封面版本说明等非正文信息。
- `body_blocks` 与有实质内容的 `appendices` 进入 `clean_text` 和检索。
- 每个 Chunk 保存 `start_char/end_char`、来源页、`section_path`、主块与重叠块 ID、`clean_text_hash` 和 `chunk_hash`。
- 270 个 Chunk 标记为带检索上下文，其中 108 个具有非零 `overlap_left`；阅读页不消费这些重叠字段。

## 表格与特殊结构

- PDF 保留源文件列宽和二维网格，不再为了规整展示机械补出空列。
- 跨页表格只在列数一致、上一页靠近页底、下一页靠近页顶时拼接，并去除重复表头。
- DOCX 读取原生表格，保留可确认的 `rowspan/colspan`。
- 无法可靠确定的二维合并关系不伪造；保留原始文本并记录 `parsing_warnings`。
- HTML 展示使用 `table_data`；向量检索使用另行生成的“表头：单元格”文本。
- 公式由解析阶段的 `formula_data/raw_text` 驱动；不能可靠转换时保留原式，不由前端根据等号猜测。

重点文件《证券公司风险控制指标计算标准规定》的 14 个分页片段恢复为 6 张逻辑表。其中附件 2 的主表跨 PDF 第 4–7 页，保持 7 列和 103 行正文；浏览器中 6 张表的含表头行数为 28、105、28、73、81、53。
