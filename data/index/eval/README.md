# 端到端问答评测集

这里的三个问题走与正式产品完全相同的链路：问题基础分析、DeepSeek 检索规划、混合检索工具调用、证据充分性判断、必要时补检、DeepSeek 回答、程序校验和独立 DeepSeek 审查。

`queries.jsonl` 不预置 Chunk ID，也不指定哪个 Chunk 必须被引用。模型从本次实际检索上下文中自行选择证据，评测器检查最终判断、适用范围、必要法规、关键限定、逐字原文、官网链接、程序校验、审查结论和循环次数。

配置 `LLM_API_KEY`、`LLM_MODEL=deepseek-v4-pro` 后运行：

```bash
pnpm eval:qa
```

本地运行会生成（不提交 Git）：

- `results.json`：保存检索计划、实际模型上下文、证据判断、最终回答、模型选择的引用、程序校验、独立审查和受限状态轨迹。
- `results.md`：按“先直接回答、后法规依据及官网链接”的格式生成简表。

这两个文件只用于本地调试并被 Git 忽略。普通前端不会展示 Chunk ID、检索排名、内部轨迹或审查过程。
